SQL学习笔记

发表于 2017-11-14 | 分类于 资料

###目录

1、基本查询语句

2、聚合函数

3、分组查询

4、联接查询

基本查询语句

select 列名列表

from 表名

where 筛选条件

order by 排序字段

模糊查询：

like：字符类型列，查询满足给定格式的数据

%:任意长度任意字符

_：长度为1的任意字符

[]：长度为1且在给定范围字符

in：各种类型列，查询等于其中任意个给定数据的数据**

between and：数值和日期类型，查询区间**

聚合函数

sum：忽略null，求和

max、min：忽略null，求最大值、最小值

avg：忽略null，求平均值

count：不会忽略null

count（*）：获取结果集中记录条数

count（字段）：获取结果集中对应字段中非空数据的个数

假如有下面这样一张表

a b c
1 1 1
null 2 2
3 null 3

count(*):3

count(b):2

count(c):3

####分组查询

select 列名列表

from 表名

where 筛选条件

groupby 分组项

having 过滤分组

orderby 排序字段

分组个数——》分组项存在不同数据的个数决定

分组之后，使用聚合函数进行统计，不是针对于所有数据，而是针对于每一组进行统计

一旦使用分组，select之后只能出现分组项和聚合函数

分组之后，需要进一步进行数据筛选，只能使用having，不能使用where

####联接查询

使用场合：当查询结果集中存在多张表的数据时，使用联接查询

分类：

内联接

外联接（左外联接、右外联接、全联接）

交叉联接

内联接：

select 列名列表

from tb_A inner join tb_B

on 联接条件

[where 筛选条件]

[groupby 分组项]

[having 过滤分组]

[orderby 排序字段]

执行流程：

求笛卡尔积

去除不满足联接条件的数据

联接结果：左表和右表存在对应满足联接条件的数据将会出现结果中

tb_A tb_B

a b c c d

1 1 1 3 3

2 2 2 4 4

3 3 3

联接条件：tb_A.c=tb_B.c

1 1 1 3 3

1 1 1 4 4

2 2 2 3 3

2 2 2 4 4

3 3 3 3 3

3 3 3 4 4

去除不满足条件数据

3 3 3 3 3

#####左外联接：

select 列名列表

from tb_A left outer join tb_B

on 联接条件

[where 筛选条件]

[groupby 分组项]

[having 过滤分组]

[orderby 排序字段]

#####执行流程：

求笛卡尔积

去除不满足联接条件的数据

保留所有左表中数据

联接结果：所有左表数据都会出现在结果中，若右表存在对应数据，则显示，否则显示null

tb_A tb_B

a b c c d

1 1 1 3 3

2 2 2 4 4

3 3 3

联接条件：tb_A.c=tb_B.c

1 1 1 3 3

1 1 1 4 4

2 2 2 3 3

2 2 2 4 4

3 3 3 3 3

3 3 3 4 4

去除不满足条件数据

3 3 3 3 3

1 1 1 null null

2 2 2 null null

右外联接：

tb_A right outer join tb_B

——》tb_B left outer join tb_A

内联接和左外联接区别：

内联接的结果<=左外联接的结果

交换两个表的位置，内联接结果不会发生改变，左外连接结果可能会发生改变

tb_A inner join tb_B——》常规情况

tb_A inner join tb_A——》特殊情况（自连接）

tb_A as A inner join tb_A as B

物理上是同一张表，但通过设置不同别名，在逻辑上分为两张表
